  <action>
    Replace the stub execution_flow in SKILL.md with the complete recursive research workflow. Keep the existing frontmatter and "What It Does" / "Execution Mode" / "CLI Tools" sections. Replace `<execution_flow>` content entirely.

    The execution_flow must contain these steps (use `<step>` XML elements):

    **Step 1: Parse Target** - Determine if target is "project" or a phase number. Set output paths:
    - Project: .planning/research/ (pass-N/ subdirs, state.json, SUMMARY.md)
    - Phase: .planning/phases/{phase-dir}/research/ (same structure)

    **Step 2: Load Prior Context** - Read PROJECT.md, ROADMAP.md, STATE.md. For phase targets, also read phase requirements from ROADMAP.md and any existing research. For project targets, this is initial research.

    **Step 3: Initialize Research State** - Call `dan-tools.cjs research init <target> --max-passes 4`

    **Step 4: Research Loop** - The core recursive loop (while convergence check says continue):

    4a. Determine dimensions for this pass:
    - Pass 1 (project or phase): ["stack", "features", "architecture", "pitfalls"] -- 4 parallel researchers (RSRCH-03, RSRCH-09)
    - Pass 2+ (any target): only dimensions with gaps from previous synthesis (RSRCH-05)

    4b. Spawn researchers -- one Task() per dimension. Each researcher gets:
    - The dimension/topic to research
    - Pass number
    - Output file path: {research-dir}/pass-{N}/{dimension}.md (or gap-{dimension}.md for pass 2+)
    - For pass 2+: the specific gap topic and reason from synthesis, plus path to prior findings
    - Project context (PROJECT.md path, phase requirements if phase-level)

    4c. Spawn synthesizer -- one Task() call. Synthesizer gets:
    - For pass 1: all pass-1 finding files
    - For pass 2+: previous synthesis.md + new gap finding files (NOT all raw files from all passes -- prevents context overload per Pitfall 2 in 03-RESEARCH.md)
    - Output path: {research-dir}/synthesis.md
    - Instruction to produce structured convergence assessment with `<gaps>` block

    4d. Read synthesis.md output. Extract gaps from `<gaps>` block and confidence from the convergence assessment table.

    4e. Record pass: `dan-tools.cjs research record-pass <target> --pass N --gaps '<gaps_json>' --confidence '<confidence_json>'`

    4f. Check convergence: `dan-tools.cjs research check-convergence <target>`. Parse should_continue. If false, exit loop.

    **Step 5: Finalize** - Copy/move synthesis.md content to the appropriate output:
    - Project: .planning/research/SUMMARY.md (plus individual STACK.md, FEATURES.md, ARCHITECTURE.md, PITFALLS.md from pass-1)
    - Phase: .planning/phases/{phase-dir}/{phase}-RESEARCH.md

    **Step 6: Update State** - Call `dan-tools.cjs state set "Status" "Research complete"` and commit via `dan-tools.cjs commit`.

    **Step 7: Report** - Display summary: passes completed, convergence reason, confidence per dimension, gaps resolved count.

    CRITICAL constraints to encode in the skill:
    - The skill is the ONLY entity that spawns agents. No agent spawns another agent (two-level hierarchy).
    - Hard cap of 4 passes is enforced by the CLI module, but also state it explicitly in the loop condition.
    - Each researcher writes to a uniquely-named file (no shared output paths).
    - Synthesizer for pass 2+ reads PREVIOUS synthesis + NEW gap findings only (not cumulative raw files).
  </action>
