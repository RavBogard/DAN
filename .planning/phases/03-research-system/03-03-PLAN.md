    - .planning/PROJECT.md (project vision and key decisions)
    - .planning/ROADMAP.md (phase goal and requirements for target phase)
    - .planning/STATE.md (current position and accumulated decisions)
    - .planning/REQUIREMENTS.md (requirement details for target phase)

    **Step 3: Identify Gray Areas** - Analyze the phase goal and requirements to identify 3-7 gray areas. Gray areas are implementation decisions the user cares about -- things where multiple valid approaches exist. Examples: "How should X be structured?", "What tradeoffs matter for Y?", "What constraints apply to Z?"

    Present gray areas as a numbered list and ask the user which ones to discuss (they can select all, some, or add their own).

    **Step 4: Discuss Each Selected Area** - For each selected gray area:
    - Ask up to 4 focused questions, one at a time
    - Questions should surface decisions, tradeoffs, constraints, and preferences
    - After 4 questions, ask if user wants to go deeper or move to next area
    - Use collaborative thinking style (not interrogation) -- reference GSD questioning.md philosophy
    - Apply scope guardrail: if user mentions something outside this phase's scope, acknowledge it, capture as deferred, redirect to current phase

    **Step 5: Write CONTEXT.md** - Produce the structured output file with these sections:
    ```markdown
    ---
    phase: {phase-number}
    created: {date}
    ---

    # Phase {N}: {Phase Name} -- Discussion Context

    ## Domain
    {Brief description of what this phase covers}

    ## Decisions
    {Numbered list of decisions made during discussion, each with:}
    - Decision: {what was decided}
    - Rationale: {why}
    - Alternatives considered: {what else was discussed}

    ## Code Context
    {Technical context relevant to implementation -- existing patterns, files, conventions}

    ## Specifics
    {Detailed implementation guidance captured from discussion}

    ## Deferred Ideas
    {Ideas mentioned but explicitly deferred -- NOT to be implemented in this phase}

    ## Open Questions
    {Questions that came up but weren't resolved -- may need research or future discussion}
    ```

    - Call `dan-tools.cjs state set "last_activity" "{date} -- Discuss phase {N}"` to update state
    - Call `dan-tools.cjs commit "docs({phase}): capture discussion context" --files {context-md-path}` to commit

    **Step 7: Report** - Summarize: number of areas discussed, decisions captured, items deferred. Suggest next step (typically `/dan:research` or `/dan:plan`).
