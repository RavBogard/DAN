    Create `bin/lib/core.cjs` with these exports:
    - `output(result, raw, rawValue)`: JSON to stdout, @file: for over 50KB (use os.tmpdir)
    - `error(msg)`: stderr + process.exit(1)
    - `atomicWriteFileSync(filePath, content)`: write to `filePath + '.tmp.' + process.pid`, then `fs.renameSync`
    - `execGit(cwd, argsArray)`: safe git execution via child_process.execFileSync (NOT execSync -- use execFileSync to avoid shell injection). Pass ['git', ...argsArray] pattern. Returns {exitCode, stdout, stderr}.
    - `toPosixPath(p)`: `p.split(path.sep).join('/')`
    - `loadConfig(cwd)`: read .planning/dan.config.json with defaults merge (mode, granularity, autonomy_level, model_profile, commit_docs, workflow flags)

    - Parse process.argv: extract --cwd, --raw flags
    - Switch on first arg to dispatch: state -> lib/state.cjs, config -> lib/config.cjs, commit -> lib/commit.cjs, frontmatter -> lib/frontmatter.cjs
    - Default case: error with usage hint

    Reference GSD patterns from research but implement true atomic writes (temp+rename) unlike GSD plain writeFileSync.
  </action>

  <action>
    Create `bin/lib/state.cjs`:
    - `handle(cwd, args, raw)`: dispatch read|get|set|patch subcommands
    - `read`: parse STATE.md using regex `**Field:** value` pattern, return all fields as JSON object. Also parse section content under ## headers.
    - `get field_name`: extract single field, return {field, value}
    - `set field_name value`: read file, regex-replace field value, atomicWriteFileSync back
    - `patch json_string`: apply multiple field updates at once
    - Use core.cjs extractField pattern from research for parsing

    Create `bin/lib/config.cjs`:
    - `handle(cwd, args, raw)`: dispatch get|set subcommands
    - Defaults: {mode: "yolo", granularity: "standard", autonomy_level: "milestone", model_profile: "quality", commit_docs: true, workflow: {research: true, plan_check: true, verifier: true, nyquist_validation: true}}
    - `get [key]`: load config, merge defaults, return full or single key
    - `set key value`: load, update, atomicWriteFileSync

    Create `bin/lib/frontmatter.cjs`:
    - `handle(cwd, args, raw)`: dispatch parse|serialize subcommands
    - `parse file_path`: read file, extract YAML between `---` markers, parse key-value/arrays/nested objects, return {frontmatter, body}
    - `serialize json body`: produce `---\nyaml\n---\nbody`
    - Support: string values, arrays (- item), nested objects (indented), booleans, numbers
    - Export parse/serialize functions directly for use by other lib modules

    - `handle(cwd, args, raw)`: parse message and files from args
    - Check commit_docs config flag; skip if false
    - Stage specified files (never `git add .`)
    - Run `git commit -m message`, handle nothing-to-commit gracefully
    - Return {committed, hash, reason}
    - Use execGit from core.cjs (which uses execFileSync, not execSync)
