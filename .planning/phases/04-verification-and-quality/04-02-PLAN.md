    2. `load_phase_context`: Read ROADMAP.md for phase goal and success criteria. Read all *-PLAN.md files in phase dir. Read all *-SUMMARY.md files. Read REQUIREMENTS.md for requirement IDs mapped to this phase.

       - Read all plan files and their must_haves (truths, artifacts, key_links)
       - Run `dan-tools.cjs verify artifacts <plan>` for each plan
       - Run `dan-tools.cjs verify key-links <plan>` for each plan

       - Check each requirement from REQUIREMENTS.md against evidence
       - Write results to `{phase_dir}/{padded}-VERIFICATION.md` using the template from 04-RESEARCH.md Pattern 4
    5. `report_results`: Read VERIFICATION.md. Report status to user. Update STATE.md with verification result.

    **dan-bugsweep/SKILL.md** -- Replace the placeholder `<execution_flow>` with the recursive loop following 04-RESEARCH.md Pattern 2. Steps:

    1. `load_phase`: Parse phase argument. Find phase dir. Read all SUMMARY.md files to collect key_files. Set MAX_CYCLES=3, cycle=0, previous_issues=[].
    2. `bugsweep_loop`: Core loop (cycle < MAX_CYCLES):
       a. Increment cycle.
       b. Spawn dan-verifier to produce fresh VERIFICATION.md.
       c. Parse VERIFICATION.md for issues list.
       d. If no issues: go to `complete`.
       e. If cycle > 1: run `dan-tools.cjs verify recurring --current '<issues_json>' --previous '<previous_json>'`. If should_escalate=true: go to `escalate`.
       f. Set previous_issues = current issues.
       g. Spawn dan-auditor with issues list and key_files.
       h. If cycle == MAX_CYCLES: go to `escalate`.
       i. Go to step (a).
    3. `escalate`: Update VERIFICATION.md bugsweep_cycles field. Update STATE.md with "Bugsweep escalated after N cycles". Report remaining issues with context to user.
    4. `complete`: Update VERIFICATION.md status=passed, bugsweep_cycles=N. Update STATE.md with "Phase N verified clean after N cycles". Report success.

    Keep skill prompts focused on orchestration logic. Do NOT embed verification logic -- reference CLI tools instead. Keep each step clear and self-contained.
