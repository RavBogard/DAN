  <behavior>
    - verifyArtifacts(cwd, planPath) reads must_haves.artifacts from plan frontmatter; for each artifact: checks file exists, checks exports array items appear in file content, checks contains string appears in file content, checks min_lines threshold; returns { all_passed, passed, total, artifacts: [{ path, exists, issues, passed }] }
    - verifyArtifacts returns passed=false with "File not found" issue when artifact path does not exist
    - verifyArtifacts returns passed=false with "Missing export: X" when export string not found in file
    - verifyArtifacts returns passed=true when file exists and all checks pass
    - verifyKeyLinks(cwd, planPath) reads must_haves.key_links from plan frontmatter; for each key_link: reads source file, tests regex pattern or substring match against target; returns { all_verified, verified, total, links: [{ from, to, via, verified, detail }] }
    - verifyKeyLinks returns verified=false with "Source file not found" when from-path missing
    - verifyKeyLinks returns verified=false with "Pattern not found" when regex does not match
    - verifyKeyLinks returns verified=true with "Pattern found" when regex matches
    - verifyPhaseCompleteness(cwd, phaseNum) uses findPhase to locate phase dir; counts PLAN.md and SUMMARY.md files; returns { complete, plan_count, summary_count, incomplete_plans }
    - verifyPhaseCompleteness returns complete=false when plans exist without matching summaries
    - verifyPhaseCompleteness returns complete=true when every plan has a summary
    - fingerprintIssue(issue) lowercases, strips "line NNN", normalizes whitespace and string literals; returns stable string
    - fingerprintIssue produces same fingerprint for same issue with different line numbers
    - detectRecurringIssues(current, previous) returns { recurring_count, total_current, ratio, should_escalate }
    - detectRecurringIssues returns should_escalate=true when ratio > 0.5
    - detectRecurringIssues returns should_escalate=false when ratio <= 0.5
    - formatVerificationReport({ phase, status, score, criteria, artifacts, keyLinks, requirements, issues, testOutput }) returns markdown string with YAML frontmatter (phase, verified, status, score, bugsweep_cycles), criteria table, artifact table, key-link table, requirements table, issues list
