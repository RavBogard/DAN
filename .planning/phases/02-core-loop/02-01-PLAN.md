<task type="auto" tdd="true">
  <name>Task 1: Plan lifecycle state machine and qualifier output parser modules</name>
  <files>bin/lib/lifecycle.cjs, bin/lib/qualify.cjs, bin/tests/test-plan-lifecycle.cjs, bin/tests/test-eq-protocol.cjs, bin/tests/test-diagnostic-routing.cjs</files>
  <behavior>
    lifecycle.cjs:
    - PLAN_STATES constant contains: DRAFT, APPROVED, IN_PROGRESS, COMPLETED, ABANDONED
    - VALID_TRANSITIONS maps each state to its allowed next states:
      - DRAFT -> [APPROVED, ABANDONED]
      - APPROVED -> [IN_PROGRESS, ABANDONED]
      - IN_PROGRESS -> [COMPLETED, ABANDONED]
      - COMPLETED -> [] (terminal)
      - ABANDONED -> [] (terminal)
    - validateTransition(from, to) returns { valid: true } or { valid: false, reason: string }
    - validateTransition rejects unknown states with descriptive error
    - getNextStates(state) returns array of valid next states
    - isTerminal(state) returns true for COMPLETED and ABANDONED

    qualify.cjs:
    - QUALIFICATION_STATUSES constant: PASS, PASS_WITH_CONCERNS, NEEDS_REVISION, FAIL
    - parseQualifierOutput(markdownText) extracts: { status, task, criteria: [{name, passed}], evidence, issues }
    - parseQualifierOutput handles the structured format from dan-qualifier agent:
      **Task:** N
      **Status:** PASS|PASS_WITH_CONCERNS|NEEDS_REVISION|FAIL (or **Grade:** as alias)
      **Criteria:** or **Criteria checklist:**
      - criterion name: PASS/FAIL
      **Evidence:** text
      **Issues:** numbered list
    - parseQualifierOutput returns { status: null, error: "..." } if status line is missing or unrecognized
    - shouldRetry(status, retryCount, maxRetries=3) returns { retry: bool, reason: string }
      - PASS -> { retry: false, reason: "passed" }
      - PASS_WITH_CONCERNS -> { retry: false, reason: "passed with concerns" }
      - NEEDS_REVISION + retryCount < 3 -> { retry: true, reason: "revision N of 3" }
      - NEEDS_REVISION + retryCount >= 3 -> { retry: false, reason: "max retries exceeded, escalate" }
      - FAIL -> { retry: false, reason: "failed, escalate" }
    - classifyFailure(taskSpec, qualifierEvidence, objectiveText) returns { classification: "intent"|"spec"|"code", reasoning: string }
      - This is a heuristic classifier using keyword/pattern matching:
      - "intent" if evidence mentions "wrong goal", "wrong problem", "not what was asked", or objective mismatch
      - "spec" if evidence mentions "ambiguous", "incomplete", "contradicts", "missing requirement", or spec issues
      - "code" as default (implementation bug)
  </behavior>
  <action>

    - Define PLAN_STATES as a frozen object with string values
    - Define VALID_TRANSITIONS as a frozen map from state to array of allowed next states

    - Implement getNextStates(state) and isTerminal(state) helpers
    - Export: PLAN_STATES, VALID_TRANSITIONS, validateTransition, getNextStates, isTerminal

    Create bin/lib/qualify.cjs:
    - Define QUALIFICATION_STATUSES as frozen object
    - Implement parseQualifierOutput(text) that uses regex to extract the structured fields from qualifier markdown output. Parse **Status:** or **Grade:** line for the status. Parse criteria items as `- name: PASS` or `- name: FAIL`. Extract evidence and issues sections.
    - Implement shouldRetry(status, retryCount, maxRetries) with the routing logic
    - Implement classifyFailure(taskSpec, qualifierEvidence, objectiveText) as a keyword-based heuristic (not LLM-based -- this is a first-pass classifier; the skill can override with LLM judgment)
    - Export: QUALIFICATION_STATUSES, parseQualifierOutput, shouldRetry, classifyFailure

    Create test files using node:test and node:assert:
    - bin/tests/test-plan-lifecycle.cjs: Test all valid transitions succeed, invalid transitions fail with reason, unknown states fail, terminal states return empty next-states, isTerminal works
    - bin/tests/test-eq-protocol.cjs: Test parseQualifierOutput with each of the 4 statuses, test with missing status line, test with Grade alias, test shouldRetry for each status at various retry counts, test maxRetries boundary
    - bin/tests/test-diagnostic-routing.cjs: Test classifyFailure returns "intent" for objective-mismatch keywords, "spec" for spec-issue keywords, "code" as default

    Follow existing test patterns from bin/tests/ (use node:test describe/it pattern with strict assertions).
